# 회사 내부 규칙 기반 연차 관리 앱 요구사항

## 1. 개요
이 앱은 법정 연차 계산기가 아닌, 회사 내부 규칙에 맞춘 개인별 연차 기록 앱입니다.
사용자는 최초 1회만 본인 기준을 설정하며, 이후 접속부터는 저장된 데이터를 자동으로 불러와 대시보드로 진입합니다.

## 2. 연차 계산 기준 항목
- 총 발생 연차
- 현재까지 실제 발생 연차
- 예상 발생 연차
- 초기 입력 사용 연차
- 이후 등록 사용 연차
- 총 사용 연차
- 남은 연차
- 다음 연차 초기화일
- 초기화일까지 남은 일수
- 주말/공휴일/휴무일 연결 사용 횟수
- 남은 연결 사용 가능 횟수
- 추천 연차 사용일

## 3. 날짜 처리 기준
- **시간대:** 모든 날짜는 한국 시간 (Asia/Seoul) 기준으로 처리.
- **포맷:** JavaScript Date 객체의 UTC 변환 이슈를 막기 위해 연차 사용일, 입사일, 휴무일 등은 모두 `YYYY-MM-DD` 형태의 문자열로 관리.
- 브라우저 locale 기준으로 공휴일을 판단하지 않음.

## 4. 휴무일 및 연결 사용 기준
계산 로직에서는 "휴무일"이라는 통합 기준을 사용합니다.
- 토요일, 일요일
- 한국 법정공휴일, 대체공휴일, 임시공휴일, 선거일, 명절(설/추석)
- 근로자의 날, 회사 창립기념일, 회사 지정 휴무일
- 사용자 수동 추가 휴무일
**주의:** 대체공휴일(`isSubstituteHoliday = true`)도 `isHoliday = true` 또는 `isNonWorkingDay = true`로 간주하여 연결 사용 계산에 포함됩니다.

## 5. 휴무일 데이터 우선순위
1. 사용자가 수동 추가/수정/삭제한 휴무일 (overrideStatus = 'added' | 'edited' | 'deleted')
2. API에서 불러온 한국 공휴일
3. localStorage에 캐시된 휴무일
4. Mock 휴무일 데이터
*수동으로 삭제한 항목은 API 업데이트 시에도 다시 나타나지 않습니다.*

## 6. 저장소 (localStorage)
- `schemaVersion`을 포함하여 버전 관리를 수행합니다.
- 저장해야 할 주요 데이터: 설정, 사용 기록, 보정값, 만근 여부, 공휴일/휴무일 데이터, override 데이터 등.
- 파싱 실패 시 앱 크래시를 방지하고 복구 안내를 제공해야 합니다.
